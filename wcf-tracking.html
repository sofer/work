Notes

It looks like there is a bit of namespace conflict between jQuery and mootools, so we are using $jpcc in place of $.

To add tracking for files, simply place

<script src="<path-to-js-files>/tracking.js" type="text/javascript"></script>

inside the head tags at any point after loading jquery.

The list of file paths is on the first line of the tracking.js file and can be amended. Use lower case only. pdf, ppt, doc, docx, pptx, xls and xlsx are all currently recognised.

We are currently using just file names to identify files. If we want the full virtual path of a document to show up in google analytics, then there is a commented-out line that can be uncommented in tracking.js. Any fully-qualified href attribute (starting "http://" or "https://") will be ignored for tracking purposes.

"One of the main advantages of the asynchronous snippet is that you can position it at the top of the HTML document. This increases the likelihood that the tracking beacon will be sent before the user leaves the page. It is customary to place JavaScript code in the section, and we recommend placing the snippet at the bottom of the section for best performance." https://support.google.com/analytics/answer/1008080
